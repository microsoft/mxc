MXC Playground — Quick Start

      
    

    

      

        
What is MXC?

        
MXC runs scripts inside isolated containers, controlling access to files, network, and desktop features.

      

      

        
Getting Started

        
    
          
  1. Choose containment — Select a containment type. Process Container is the default and supports pre-made scenarios.
    2. 
          
  2. Pick a runtime — cmd.exe, PowerShell, Python, Custom, or MXC JSON Config. Not installed? Follow the install instructions shown.
    3. 
          
  3. Select a test — Pre-made scenarios auto-configure policy. ✓ = should succeed, ✗ = should be blocked.
    4. 
          
  4. Run — Click "▶ Run in Sandbox" to run one test, or "▶▶ Run All Tests" to run all tests for the selected runtime.
    5. 
        

      

      

        
Runtime Installation

        
    
          
  • 🐍 Python — Install via winget install Python.Python.3.14. Works out of the box.
    • 
          
  • 🟦 PowerShell 7+ — Supports MSI package installed version. MSIX and Microsoft Store versions are not yet supported. Install MSI package →
    • 
        

      

      

        
Containment Types

        
    
          
  • 🛡️ Base Process Container — Default for 0.5.0-dev. Uses CreateProcessInSandbox. Supports all pre-made scenarios.
    • 
          
  • 🛡️ AppContainer — Default for 0.4.0-alpha. Standard Windows AppContainer isolation.
    • 
          
  • 🧪 Windows Sandbox / MicroVM / WSLC — Experimental. Select from the Containment dropdown — these only support MXC JSON Config mode (paste your own config).
    • 
        

      

      

        
Top Bar Controls

        
    
          
  • 🧪 Experimental — On by default. Passes --experimental to wxc-exec to enable experimental features.
    • 
          
  • 🐛 Debug Mode — Shows detailed wxc-exec output.
    • 
          
  • 👁️ Developer View — Live Sandbox Policy and Container Config JSON on the right.
    • 
        

      

      

        
Notes

        
    
          
  • 📝 Scripts are inline commands. For .ps1/.py files, use full paths.
